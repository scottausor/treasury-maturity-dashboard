# Treasury MSPD Scraper

## Overview
Automated workflow that downloads the latest Treasury Department Monthly Statement of Public Debt (MSPD), parses all maturity dates for bills, notes, and bonds, and exports aggregated data to JSON for use in analytics dashboards.

## What It Does
1. **Downloads** the latest MSPD PDF from `treasurydirect.gov`
2. **Extracts** all Treasury Bill, Note, and Bond maturity dates from the PDF
3. **Aggregates** maturities by month and year
4. **Exports** clean JSON with summary stats and detailed breakdowns
5. **Cleans up** temporary files

## Output Data
**Canonical file:** `treasury_maturity_latest.json`

Contains:
- **Summary totals** (total marketable, bills, notes, bonds)
- **Monthly breakdown** (all instruments) for next 12 months
- **Yearly breakdown** for next 30 years
- **Snapshot date** of when data was extracted

Example structure:
```json
{
  "date": "2026-02-28",
  "summary": {
    "total_marketable": 30616419,
    "bills_total": 6808106,
    "notes_total": 15767155,
    "bonds_total": 5298564,
    "snapshot_date": "February 28, 2026"
  },
  "monthly": [
    {"month": "Mar 26", "bills": 347, "notes": 241, "bonds": 12, "total": 600},
    ...
  ],
  "yearly": [
    {"year": "2026", "maturity": 7206},
    ...
  ]
}
```

## Error Handling
- **PDF not found:** Script fails gracefully and reports URL not found
- **Parsing errors:** Partial data is still saved; check logs for which instruments failed
- **Network timeout:** Retries once after 10 second wait
- **Temp file cleanup:** Always runs, even on failure

## Troubleshooting

### "Could not fetch MSPD PDF"
- Treasury's website may have changed URL
- Check `https://treasurydirect.gov/NP/debt/current` manually
- Look for latest MSPD PDF link and update script if needed

### "No data extracted"
- PDF structure may differ from expected format
- Check PDF opens manually in a reader
- Review console logs for which section parsing failed

### "JSON not created"
- Check folder permissions: `C:\Users\scott\OneDrive\Documents\Research\My Workflow\`
- Ensure write access to the directory
- Check disk space available

### Verify Success
After running, check:
1. File exists: `treasury_maturity_latest.json`
2. File size > 1KB (should be ~2-3KB)
3. Contains JSON with `"monthly"` and `"yearly"` keys
4. Exit code = 0

## Updating the Dashboard
The React dashboard reads from this JSON file:
- If hosting on GitHub: update the raw GitHub URL in dashboard
- If local: point dashboard to `treasury_maturity_latest.json` directly
- Refresh dashboard after run to see latest data

## Notes
- Treasury releases MSPD on ~4th business day of month
- Script runs on 5th to ensure latest PDF is available
- Large PDFs (>100 pages) may take 1 minute to parse
- Maturity dates are as of statement date, not current date
